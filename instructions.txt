PROJECT BLUEPRINT: 3D INTERACTIVE PORTFOLIO

1. PROJECT CONTEXT & OBJECTIVES
-------------------------------
- Goal: Build a high-fidelity personal portfolio with 3D animations.
- Target Audience: Recruiters and technical hiring managers.
- Hosting: GitHub Pages (Static hosting).
- Version Control: Git (Managed via Git Extensions).
- Primary Constraint: Must be a Client-Side Rendered (CSR) Single Page Application.
- Personal Brand: Java Backend Developer specializing in scalable microservices, seeking to showcase full-stack capabilities.

2. TECHNICAL STACK DEFINITION
-----------------------------
The AI Agent should utilize the following technologies:
- Core: React 18+ (via Vite).
- Language: JavaScript (ES6+).
- Styling: Tailwind CSS (for layout) + Custom CSS (for complex animations).
- Animation Engine: Framer Motion (Physics & Transitions).
- Visual Component Library: React Bits (Micro-interactions & 3D Backgrounds).
- Build/Deploy: `gh-pages` (npm package) for GitHub Pages deployment.

3. IMPLEMENTATION ROADMAP
-------------------------

PHASE A: INITIALIZATION & DEPENDENCIES
Execute the following commands to scaffold the project:

  # 1. Initialize Vite Project
  npm create vite@latest portfolio-amir -- --template react

  # 2. Install Core Dependencies
  cd portfolio-amir
  npm install

  # 3. Install Design & Animation Dependencies
  npm install framer-motion tailwindcss postcss autoprefixer sass
  npm install gh-pages --save-dev

  # 4. Initialize Tailwind
  npx tailwindcss init -p

PHASE B: ARCHITECTURE & FOLDER STRUCTURE
The project must adhere to this strict directory structure:

  src/
  ├── assets/              # Images and Icons
  ├── components/
  │   ├── UI/              # Standard atomic components (Buttons, Cards)
  │   └── Visuals/         # REACT BITS components (e.g., Ballpit.jsx)
  ├── sections/            # Semantic Page Sections (Hero, About, Projects)
  ├── data/                # Static Data Files (JSON/JS objects)
  ├── hooks/               # Custom Hooks (useScroll, etc.)
  ├── App.jsx
  ├── main.jsx
  └── index.css
  public/
  └── resume.pdf           # Static resume file

PHASE C: CONFIGURATION FILES

1. Tailwind Configuration (tailwind.config.js):
   Configure content paths to include all source files.
   
   export default {
     content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx}"],
     theme: { extend: {} },
     plugins: [],
   }

2. CSS Entry Point (src/index.css):
   Inject Tailwind directives.
   
   @tailwind base;
   @tailwind components;
   @tailwind utilities;

3. Vite Configuration (vite.config.js):
   CRITICAL: The `base` property must match the GitHub repository name.
   
   import { defineConfig } from 'vite'
   import react from '@vitejs/plugin-react'
   
   // REPLACE 'repository-name' WITH THE ACTUAL GITHUB REPO NAME
   export default defineConfig({
     plugins: [react()],
     base: '/repository-name/',
   })

4. Package.json Scripts:
   Add deployment scripts.
   
   "homepage": "https://<your-username>.github.io/<repository-name>",
   "scripts": {
     "dev": "vite",
     "build": "vite build",
     "predeploy": "npm run build",
     "deploy": "gh-pages -d dist"
   }

PHASE E: DEPLOYMENT WORKFLOW
----------------------
Since the user utilizes Git Extensions:
1. Commit changes to the `main` branch locally.
2. Push `main` to GitHub.
3. Run `npm run deploy` in the terminal.
   (This builds the project to a `dist` folder and pushes ONLY that folder to the `gh-pages` branch).
4. GitHub Pages serves the site from the `gh-pages` branch automatically.

NOTE ON GITIGNORE:
The user should select the "Node" template when creating the repository on GitHub.
It correctly excludes:
- node_modules/
- dist/
- .env
- logs
